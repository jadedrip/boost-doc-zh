内部提升策略(Internal Promotion Policies)

通常，在以float精度计算一个函数的时候，在函数内部通过构造一个double精度的计算来确保精度，并将结果进行舍入： 有两个策略来影响内部的提升是否进行：

策略

含义

boost::math::policies::promote_float<B>

指示在函数内部， float 参数是否被提升为 double 精度：缺省为boost::math::policies::promote_float<true>

boost::math::policies::promote_double<B>

指示在函数内部， double 参数是否被提升为 long double 精度：缺省为boost::math::policies::promote_double<true>

例子

假设我们想要函数tgamma 在计算时不会提升为long double精度，那么我们应当使用：

#include <boost/math/special_functions/gamma.hpp>

using namespace boost::math::policies;
using namespace boost::math;

// 定义一个策略:
typedef policy<
      promote_double<false> 
      > my_policy;
      
// 调用这个函数:
double t1 = tgamma(some_value, my_policy());

// 另一方面，我们可以使用函数 make_policy 在调用点定义所需要的策略:
double t2 = tgamma(some_value, make_policy(promote_double<false>()));

另一方面，假设我们想要一个分布在进行计算时不进行从float 到 double的精度提升，那么我们应该使用：

#include <boost/math/distributions/normal.hpp>

using namespace boost::math::policies;
using namespace boost::math;

// 定义策略:
typedef policy<
      promote_float<false>
      > my_policy;

// 定义分布:
typedef normal_distribution<float, my_policy> my_norm;

// 获取分位点:
float q = quantile(my_norm(), 0.05f);
