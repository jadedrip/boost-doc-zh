错误处理策略(Error Handling Policies)

错误处理有两个正交的方面(orthogonal aspects)：

  • 如何处理错误 (如果有的话)。
  • 产生的是哪一种错误。
当发生错误时可用的操作

如何处理错误由一个枚举类型包装起来：

namespace boost { namespace math { namespace policies {

enum error_policy_type
{
   throw_on_error = 0, // 输出异常.
   errno_on_error = 1, // 设置 ::errno & 返回 0, NaN, infinity or best guess.
   ignore_error = 2, // 返回 0, NaN, infinity or best guess.
   user_error = 3  // 调用一个用户定义的错误处理方法.
};

}}} // namespaces

这些枚举值具有下面的含义：

throw_on_error

依据错误的类型，将会抛出下面的异常之一：

错误类型

异常

定义域错误(Domain Error )

std::domain_error

Pole Error

std::domain_error

溢出错误(Overflow Error)

std::overflow_error

下溢错误(Underflow Error)

std::underflow_error

Denorm Error

std::underflow_error

计算错误(Evaluation Error)

boost::math::evaluation_error

未定义结果错误(Indeterminate Result Error)

std::domain_error

errno_on_error

依据错误的类型，将会设置全局变量::errno 为以下值之一，如果这种错误被忽略的话则返回相同的值。

未定义结果错误(Indeterminate Result Error)

依赖于错误所发生的函数
user_error

将会调用 一个用户定义的错误处理方法：这些错误处理方法在 boost/math/policies/error_handling.hpp进行了前向声明(forward declaration)，但实际的定义必须由用户提供：

namespace boost{ namespace math{ namespace policies{

template <class T>
T user_domain_error(const char* function, const char* message, const T& val);

template <class T>
T user_pole_error(const char* function, const char* message, const T& val);

template <class T>
T user_overflow_error(const char* function, const char* message, const T& val);

template <class T>
T user_underflow_error(const char* function, const char* message, const T& val);

template <class T>
T user_denorm_error(const char* function, const char* message, const T& val);

template <class T>
T user_rounding_error(const char* function, const char* message, const T& val);

template <class T>
T user_evaluation_error(const char* function, const char* message, const T& val);

template <class T>
T user_indeterminate_result_error(const char* function, const char* message, const T& val);

}}} // namespaces

注意参数function 和 message 可能包含"%1%" 格式化分类符号(format specifiers) 来与 Boost.Format库联合使用。如果这些字符串将会呈现给程序的最终用户，那么"%1%" 格式化分类符号(format specifiers) 将使用类型T的名字在function 字符串中进行替换，如果有一个 %1% 分类符号在message 字符串中，那么它应当使用val的值进行替换。

在指南中有更多关于用户自定义错误处理方法的信息。

产生的错误类型(Kinds of Error Raised)

由这个库报告的错误类型有6种，总结在下面的表中：

错误类型

策略类

描述

定义域错误(Domain Error)

boost::math::policies::domain_error<action>

当有一个或多个参数超出函数的定义域时发生。

缺省为：boost::math::policies::domain_error<throw_on_error>

如果操作设置为throw_on_error 那么就抛出 std::domain_error 异常

Pole Error

boost::math::policies::pole_error<action>

当有一个或多个参数导致函数在一个pole处计算的时候发生

缺省为：boost::math::policies::pole_error<throw_on_error>

如果操作设置为throw_on_error 那么就抛出std::domain_error 异常

溢出错误 (Overflow Error)

boost::math::policies::overflow_error<action>

当计算的结果太大而无法在所使用的浮点类型中表示的时候发生

缺省为：boost::math::policies::overflow_error<throw_on_error>.

如果操作设置为throw_on_error 那么就抛出std::overflow_error. 异常

下溢错误(Underflow Error)

boost::math::policies::underflow_error<action>

当 计算的结果太小而无法在所使用的浮点类型中表示的时候发生

缺省为：boost::math::policies::underflow_error<ignore_error>

如果操作设置为throw_on_error 那么就抛出std::underflow_error 异常

Denorm Error

boost::math::policies::denorm_error<action>

当函数的返回值是 denormalised value时发生.

缺省为：boost::math::policies::denorm_error<ignore_error>

如果操作设置为throw_on_error 那么就抛出std::underflow_error 异常

舍入错误(Rounding Error)

boost::math::policies::rounding_error<action>

当 round, trunc 或 modf 中之一使用一个没有整数表示的参数调用时，或太大以至于不能在结果类型中表示的时候发生

缺省为：boost::math::policies::rounding_error<throw_on_error>

如果操作设置为throw_on_error 那么就抛出boost::math::rounding_error 异常

计算错误(Evaluation Error)

boost::math::policies::evaluation_error<action>

当函数的返回值是定义良好的(well defined)并且是有限值，但是我们无法计算这个值时发生。通常这会在一个迭代方法不能收敛时发生。当然，理想情况下这种错误永远不应该发生：如果存在这个错误，请报告这个bug!

缺省为：boost::math::policies::evaluation_error<throw_on_error>

如果操作设置为throw_on_error 那么就抛出boost::math::evaluation_error 异常

未定义结果错误(Indeterminate Result Error)

boost::math::policies::indeterminate_result_error<action>

当对于传递给函数的参数值，函数的结果是未定义的时候发生。

缺省为：boost::math::policies::indeterminate_result_error<ignore_error>

如果操作设置为throw_on_error 那么就抛出std::domain_error 异常

例子

假设我们想要函数tgamma 以一种以兼容( C-compatible)的方式运行，并设置全局变量::errno 而不是抛出一个异常，那我们可以在函数的调用点通过使用下面的方法来达到：

#include <boost/math/special_functions/gamma.hpp>

using namespace boost::math::policies;
using namespace boost::math;

// 定义一个策略:
typedef policy<
      domain_error<errno_on_error>, 
      pole_error<errno_on_error>,
      overflow_error<errno_on_error>,
      policies::evaluation_error<errno_on_error> 
      > my_policy;
      
// 调用函数:
double t1 = tgamma(some_value, my_policy());

// 另一方面我们可以使用函数 make_policy 并在调用点定义所有的变量 :
double t2 = tgamma(some_value, make_policy(
         domain_error<errno_on_error>(), 
         pole_error<errno_on_error>(),
         overflow_error<errno_on_error>(),
         policies::evaluation_error<errno_on_error>() 
      ));

假设我们想要一个统计分布类型返回无限值，而不是抛出异常，那么我们可以使用： 

#include <boost/math/distributions/normal.hpp>

using namespace boost::math::policies;
using namespace boost::math;

// 定义一个策略:
typedef policy<
      overflow_error<ignore_error>
      > my_policy;
      
// 定义分布对象:
typedef normal_distribution<double, my_policy> my_norm;

// 获得一个分位点:
double q = quantile(my_norm(), 0.05);
