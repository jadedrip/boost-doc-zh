在名字空间作用域设置策略(Setting Polices at Namespace Scope)

有时，你真正想做的是将所有的特殊函数或分布类型引入到一个特定的名字空间作用域中， 并且让它们使用一个特定的策略。有两个宏来辅助这一点的实现：

BOOST_MATH_DECLARE_SPECIAL_FUNCTIONS(Policy)

以及：

BOOST_MATH_DECLARE_DISTRIBUTIONS(Type, Policy)

你可以在任何的特殊函数或分布类型的头文件#include后面使用这两个宏中的任何一个。例如：

#include <boost/math/special_functions/gamma.hpp>

namespace myspace{

using namespace boost::math::policies;

// 定义一个在溢出时不抛出异常的策略:
typedef policy<overflow_error<errno_on_error> > my_policy;

// 定义在这个作用域中的特殊函数使用这个策略:   
BOOST_MATH_DECLARE_SPECIAL_FUNCTIONS(my_policy)

}

//
// 现在我们可以使用 myspace::tgamma .
// 它们将会自动地使用"my_policy":
//
double t = myspace::tgamma(30.0); // 在溢出错误时不会抛出异常

在这个例子中，使用 BOOST_MATH_DECLARE_SPECIAL_FUNCTIONS 产生的结果是一系列小的内联前向函数(forwarding function)函数得到定义：

template <class T>
inline T tgamma(T a){ return ::boost::math::tgamma(a, mypolicy()); }

template <class T>
inline T lgamma(T a) ( return ::boost::math::lgamma(a, mypolicy()); }

以及其它的函数。注意尽管对于所有的特殊函数都定义了这些前向函数(forwarding function)，然而，除非你包含了这些你所使用的特殊函数的头文件(或者使用boost/math/special_functions.hpp 来包含所有的类和函数)，对于那些前向函数(forwarding function)，你将会遇到一个编译时的错误：已经声明，但是没有定义。

我们可以对分布类型进行相同的处理，但这一次我们需要指定使用的浮点类型：

#include <boost/math/distributions/cauchy.hpp>

namespace myspace{

using namespace boost::math::policies;

// 定义一个将要使用的策略，在这种情况下，我们想要所有的分布
// 的访问函数都可以编译， 虽然它们在数学上是未定义的
typedef policy<assert_undefined<false> > my_policy;

BOOST_MATH_DECLARE_DISTRIBUTIONS(double, my_policy)

}

// 现在我们可以使用 myspace::cauchy ，它将会使用
// 策略myspace::mypolicy:
//
// 这可以通过编译，但是会在运行时产生一个定义域错误:
//
void test_cauchy()
{
   try
   {
      double d = mean(myspace::cauchy());
   }
   catch(const std::domain_error& e)
   {
      std::cout << e.what() << std::endl;
   }
}

在这个例子中， BOOST_MATH_DECLARE_DISTRIBUTIONS 的作用是为每一个分布类型定义一个类似于下面的 typedef ： 

typedef boost::math::cauchy_distribution<double, my_policy> cauchy;
tyepdef boost::math::gamma_distribution<double, my_policy> gamma;

以及其它的typedef。每个typedef的名字是分布类型的名字去掉"_distribution" 后缀之后的名字。
