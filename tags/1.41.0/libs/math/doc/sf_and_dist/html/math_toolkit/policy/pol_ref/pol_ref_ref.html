策略类参考(Policy Class Reference)

在这里没有太多的要说，policy 类只是一系列的策略的编译时的包装容器：

policy 类的成员 typedef 打算作为库的内部使用，但在这里出于完整性的考虑而进行了简要的说明。

policy<...>::domain_error_type

指定如何处理定义域错误，是模板参数为domain_error的boost::math::policies::domain_error<> 的一个实例，domain_error是error_policy_type 的枚举值之一。

policy<...>::pole_error_type

指定如何处理pole-errors，是模板参数为pole_error的boost::math::policies::pole_error<> 的一个实例， pole_error是error_policy_type 的枚举值之一。

policy<...>::overflow_error_type

指定如何处理溢出错误，是模板参数为overflow_error 的 boost::math::policies::overflow_error<> 的一个实例，overflow_error 是error_policy_type 的枚举值之一。

policy<...>::underflow_error_type

指定如何处理下溢错误，是模板参数为underflow_error的boost::math::policies::underflow_error<> 的一个实例，underflow_error 是 error_policy_type 的枚举值之一

policy<...>::denorm_error_type

指定如何处理 denorm errors，是模板参数为denorm_error的boost::math::policies::denorm_error<> 的一个实例，denorm_error是error_policy_type 的枚举值之一。

policy<...>::rounding_error_type

指定如何处理舍入错误，是模板参数为rounding_error的boost::math::policies::rounding_error<> 的一个实例，rounding_error是error_policy_type 的枚举值之一。

policy<...>::evaluation_error_type

指定如何处理计算错误，是模板参数为evaluation_error的boost::math::policies::evaluation_error<> 的实例之一，evaluation_error是error_policy_type 的枚举值之一。

policy<...>::indeterminate_error_type

指定如何处理未定义结果错误，是模板参数为indeterminate_result_error的boost::math::policies::indeterminate_result_error<>的一个实例， indeterminate_result_error是error_policy_type 的枚举值之一。

policy<...>::precision_type

指定内部精度来使用二进制数字( binary digits )(使用0来表示无论内部精度是多少)。是boost::math::policies::digits2<N> 的一个实例，这个类反过来又继承自boost::mpl::int_<N>。

policy<...>::promote_float_type

指定是否在内部进行从float 到double 精确度的提升。是boost::math::policies::promote_float<B> 一个实例，这个类反过来又继承自boost::mpl::bool_<B>。

policy<...>::promote_double_type

指定是否在内部进行从double 到long double 精确度的提升。是boost::math::policies::promote_float<B> 一个实例，这个类反过来又继承自boost::mpl::bool_<B>. 

policy<...>::discrete_quantile_type

指定如何计算离散分位点，是使用discrete_quantile_policy_type值之一实例化的 boost::math::policies::discrete_quantile<> 的一个实例。

policy<...>::assert_undefined_type

指定数学上未定义的性质是否设定为编译时错误，或作为运行时错误。是boost::math::policies::assert_undefined<B> 一个实例，这个类反过来又继承自boost::math::mpl::bool_<B>. 

template <...argument list...>
typename normalise<policy<>, A1>::type make_policy(...argument list..);

make_policy 是一个辅助函数，用于将一系统的策略转换为正规化的(normalised)的 policy 类。

template <class Policy, 
          class A1 = default_policy, 
          class A2 = default_policy, 
          class A3 = default_policy,
          class A4 = default_policy,
          class A5 = default_policy,

          class A9 = default_policy,
          class A10 = default_policy,
          class A11 = default_policy,
          class A12 = default_policy,
          class A13 = default_policy>
struct normalise
{
   typedef computed-from-template-arguments type;
};

normalise 类模板将一个policy 类的实例转化为正常的形式(normalised form)。这在内部用于减少代码膨胀：因此实例化一个使用policy<A,B> 策略或policy<B,A> 策略的特殊函数实际上生成了相同的代码。

更进一步，normalise 可以用来使用一个或多个策略来合并成一个策略：例如，许多特殊函数在实际的函数实现之前 使用这个类来将它们不使用的策略设置为缺省的策略。通过这种方式来避免了代码膨胀。因为实际的实现依赖于实际使用的策略类型。
