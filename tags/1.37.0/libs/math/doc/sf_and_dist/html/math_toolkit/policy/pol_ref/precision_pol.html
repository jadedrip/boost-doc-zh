精度策略(Precision Policies)

有两个等价的策略来影响计算结果时的精度，这些策略缺省为0－意思是以这个使用的类型的 最大精度来进行计算－但也可以设置为其它的值来降低计算时所使用的精度。

namespace boost{ namespace math{ namespace policies{

template <int N>
digits10;

template <int N>
digits2;

}}} // namespaces

就如你可能想到的那样，digits10 指定使用的十进制数字，digits2 指定二进制数字位数。在内部实现中，无论使用哪一个，精度都被转化为二进制数字位数。

这个策略在编译时设定，因为大多数的特殊函数都是在编译时依据所要求的精度以及使用的数值类型来选择合适的逼近方法。

例如，我们可以将函数tgamma的计算设为5个十进制数字，使用：

#include <boost/math/special_functions/gamma.hpp>

using namespace boost::math;
using namespace boost::math::policies;

typedef policy<digits10<5> > pol;

double t = tgamma(12, pol());

或者使用make_policy：

#include <boost/math/special_functions/gamma.hpp>

using namespace boost::math;
using namespace boost::math::policies;

double t = tgamma(12, policy<digits10<5> >());

并对一个分布的分位点近似到25个十进制数字：

#include <boost/math/distributions/normal.hpp>

using namespace boost::math;
using namespace boost::math::policies;

double q = quantile(
      normal_distribution<double, policy<digits2<25> > >(), 
      0.05);
