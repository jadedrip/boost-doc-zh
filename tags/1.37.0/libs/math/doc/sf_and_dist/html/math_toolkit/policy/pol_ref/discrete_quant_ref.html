离散分位点函数(Discrete Quantile Policies)

如果一个统计分布是离散的，那么随机变量的值只能是整数－当计算分位点时， 这给我们留下了一个问题－我们或者忽略分布的离散性并返回一个浮点数， 或者我们可以将结果舍入为一个整数。就像实际发生的那样， 计算整数值要比计算浮点值要快很多，所以，返回整数值有明显的优点， 但是我们需要确定如何对结果进行舍入。discrete_quantile 策略定义了分位点函数如何工作，以及结果如何进行舍入。

enum discrete_quantile_policy_type { real, integer_round_outwards, // 缺省

integer_round_inwards, integer_round_down, integer_round_up, integer_round_nearest }; template <discrete_quantile_policy_type> struct discrete_quantile;

discrete_quantile 可以接收的值具有以下含义：

real

忽略分布的离散性并返回浮点结果。例如：

#include <boost/math/distributions/negative_binomial.hpp>

using namespace boost::math;
using namespace boost::math::policies;

typedef negative_binomial_distribution<
      double, 
      policy<discrete_quantile<integer_round_inwards> > 
   > dist_type;
   
// 下分位点:
double x = quantile(dist_type(20, 0.3), 0.05);
// 上分位点:
double y = quantile(complement(dist_type(20, 0.3), 0.05));

结果为x = 27.3898 且 y = 68.1584.

integer_round_outwards

这是缺省的策略：返回一个整数值使得：

  • 下分位点 (概率小于0.5)向下舍入
  • 上分位点 (概率大于0.5)向上舍入

这通常是最安全的舍入策略，因为它保证单侧(one-sided)或双侧区间(two-sided)至少包含要求的覆盖范围。例如：

#include <boost/math/distributions/negative_binomial.hpp>

using namespace boost::math;

// 下分位点向下舍入:
double x = quantile(negative_binomial(20, 0.3), 0.05);
// 上分位点向上舍入:
double y = quantile(complement(negative_binomial(20, 0.3), 0.05));

结果为x = 27 ( 27.3898向下舍入) 且 y = 69 (68.1584向上舍入)。

随机变量x和y定义为满足：

cdf(negative_binomial(20), x) <= 0.05
cdf(negative_binomial(20), y) >= 0.95

换句话说，我们可以保证在中间区域至少有90%的覆盖，并且在每个尾部范围内的覆盖不超过5%。

integer_round_inwards

这是integer_round_outwards对立面： 返回一个整数值，使得满足：

  • 下分位点 (概率小于0.5)向上舍入
  • 上分位点 (概率大于0.5)向下舍入

例如：

#include <boost/math/distributions/negative_binomial.hpp>

using namespace boost::math;
using namespace boost::math::policies;

typedef negative_binomial_distribution<
      double, 
      policy<discrete_quantile<integer_round_inwards> > 
   > dist_type;
   
// 下分位点向上舍入:
double x = quantile(dist_type(20, 0.3), 0.05);
// 上分位点向下舍入:
double y = quantile(complement(dist_type(20, 0.3), 0.05));

结果为x = 28 ( 27.3898向上舍入) 且 y = 68 (68.1584向下舍入)。

随机变量x和y定义为满足：

cdf(negative_binomial(20), x) >= 0.05
cdf(negative_binomial(20), y) <= 0.95

换句话说，我们可以保证在中间区域不超过有90%的覆盖，并且在每个尾部范围内的覆盖不少于5%。

integer_round_down

不管是上分位点还是下分位点，永远向下舍入为一个整数。

integer_round_up

不管是上分位点还是下分位点，永远向上舍入为一个整数。

integer_round_nearest

永远将结果舍入为最接近的的整数值，不管是上分位点还是下分位点。这会在平均情况下产生要求的覆盖范围，但对于特殊的例子可能导致明显的或多或少的与要求的差别。例如：

#include <boost/math/distributions/negative_binomial.hpp>

using namespace boost::math;
using namespace boost::math::policies;

typedef negative_binomial_distribution<
      double, 
      policy<discrete_quantile<integer_round_nearest> > 
   > dist_type;
   
// 下分位点向上舍入:
double x = quantile(dist_type(20, 0.3), 0.05);
// 上分位点向下舍入:
double y = quantile(complement(dist_type(20, 0.3), 0.05));

结果为x = 27 (由27.3898舍入) 而 y = 68 (由 68.1584舍入)。
