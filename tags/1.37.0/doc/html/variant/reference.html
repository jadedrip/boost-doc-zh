BoundedType

有关 有界类型 的要求如下：

  • 可复制构造 [20.1.3]。
  • 析构函数支持无抛出的异常安全保证。
  • 在 variant 模板实例化处，类型是完整的。(关于如何用类型包装器接受不完整类型以实现递归 variant 类型，请见 boost::recursive_wrapper<T>)。

被指定作为 variant 的模板参数的类型必须至少满足以上要求。此外，只有在其有界类型满足以下额外要求时，variant 的某些特性才可用：

  • 可赋值：variant 本身为 可赋值的 当且仅当它的每一个有界类型都满足该概念的要求。(注意，最外层的 const-限定 类型以及引用类型不能满足这些要求)
  • 可缺省构造 [20.1.4]：variant 本身是 可缺省构造的 当且仅当它的第一个有界类型(即 T1)满足该概念的要求。
  • 相等性可比较：variant 本身是 相等性可比较的 当且仅当它的每一个有界类型都满足该概念的要求。
  • 小于关系可比较：variant 本身是 小于关系可比较的 当且仅当它的每一个有界类型都满足该概念的要求。
  • 可流输出：variant 本身是 可流输出的 当且仅当它的每一个有界类型都满足该概念的要求。

StaticVisitor

例子

对于类型 T 的 静态访问者 的要求如下：

  • 必须通过重载 operator() 以支持象函数那样的调用，明确接受类型为 T 的任意值。
  • 必须提供内嵌类型 result_type. (有关如何将函数作为访问者使用，请见 boost::visitor_ptr)
  • 如果 result_type 不是 void, 则该函数对象的每个操作必须返回一个可以隐式转换为 result_type 的值。
例子

以下类满足作为多个类型的静态访问者的要求(如，显式的：int 和 std::string; 或者隐式的：short 和 const char *; 等等)：

class my_visitor
    : public boost::static_visitor<int>
{
public:

    int operator()(int i)
    {
        return i * 2;
    }

    int operator()(const std::string& s)
    {
        return s.length();
    }

};

以下是另一个例子，它的函数调用操作符是一个模板成员函数，可以操作多种类型的值。因此，以下类可作为任意支持流输出的类型的访问者(如 int, double, std::string, 等等)：

class printer
    : public boost::static_visitor<>
{
    template <typename T>
    void operator()(const T& t)
    {
        std::cout << t << std::endl;
    }
};

OutputStreamable

对 可流输出 类型 T 的要求如下：

  • 对于类型 T 的任意对象 t, std::cout << t 必须是有效表达式。
